How EvoDoc Works

EvoDoc ingests a structured symptom checklist, optional notes, and known risk factors. We transform this into features (mostly sparse binaries) and run fast, calibrated models to produce a transparent **Top-k differential** with rationale chips.

Workflow

  1. Intake symptoms (patient or MA). Structured + free-text.
  2. Feature build: binary symptom vector, derived flags (e.g., fever+rigors), optional embeddings from notes.
  3. Model pass: gradient boosting / linear baselines; calibrated probabilities.
  4. Explain: show contributing features and safe-guards (red-flags, interactions).
  5. Document: copyable snippet + audit trail for supervision.

Safety

  • Not a diagnosis. Designed for **clinician-in-the-loop** use only.
  • Soft rules for red-flag escalation and medication interactions preview.
  • Model cards and drift monitoring before any real deployment.
